# Contributing to Kino

Thanks for your interest in contributing. Kino is a local-first personal finance desktop app built with Go + Wails + Vue 3 + SQLite. All data lives in a `.kino` file on the user's machine — no servers, no accounts.

## Dev setup

**Prerequisites:** Go 1.24+, Node 18+, [Wails v2 CLI](https://wails.io/docs/gettingstarted/installation)

```bash
# Install Wails CLI (one time)
go install github.com/wailsapp/wails/v2/cmd/wails@latest

# Frontend dependencies
cd frontend && npm install && cd ..

# Live development (hot reload, Go + Vue)
wails dev

# Production build
wails build
```

The dev server also starts a browser-accessible UI at `http://localhost:34115` with all Go methods callable from devtools.

## Code conventions

- **Cents everywhere.** All monetary values are `int64` cents. Never introduce `float64` for money.
- **No external calls.** Kino is intentionally offline-first. Don't add HTTP calls to external services.
- **App struct = RPC boundary.** Public methods on `App` in `app*.go` are auto-exposed as JS via Wails. Return types must be JSON-serializable; errors go in the second return value.
- **Don't edit generated bindings.** `frontend/src/wailsjs/` is auto-generated by `wails dev` / `wails build`. Run the tool, don't hand-edit.
- **Migrations are idempotent.** All schema changes go through `internal/db/` with a `schema_version` check. Increment the version constant.
- **Transfer pairs stay in sync.** A transfer between two accounts is two linked rows joined by `TransferPairID`. Both sides must be created/updated/deleted together.
- Keep diffs focused — avoid drive-by refactors or touching unrelated files.

See [`AGENTS.md`](AGENTS.md) for the full architecture map, module breakdown, and conventions.

## Workflow

1. **Open an issue first** for non-trivial changes (new features, schema changes, anything touching the import pipeline or transfer logic). Typo fixes can skip straight to a PR.
2. **Branch + PR** against `main`. Describe what changed, why, and how you tested it.
3. **Sign off your commits** with `git commit -s` — every commit needs a `Signed-off-by:` trailer ([Developer Certificate of Origin](https://developercertificate.org)). PRs without sign-off will be asked to amend.
4. **Update `AGENTS.md`** if you add a new module, change architecture, or introduce new conventions.
